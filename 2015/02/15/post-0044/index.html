Tiger Team 养成之路(17) - iOS开发之障碍解决

本系列的目的

除产品技能外，让我同样掌握移动端设计和开发技能，完成整个产品周期

本系列的目标

iOS端RSS阅读类App，对象是我的博客hp256.com

用了2个番茄钟，解决了上2个番茄钟遇到的阻碍

继续阅读「Programming in Objective-C 6th Edition」至236页，解决了第9章，第10章，第11章遇到了阻碍，采用的方式是

终点知识点做笔记，把可能未来会用到的内容加标注页号码。

今天的笔记如下

polymorphism
不同的类class可以用同样的方法method名字

dynamic typing
直到程序执行时才决定某个对象属于什么类。

dynamic binding
直到程序执行时才决定调用对象的哪个方法。

见P180

自己写Init 见P199

加一个_在实例的变量名之前是一个广泛的做法 比如
@synthesize window = _window;
it says to synthesize the getter and setter for the property named window and to associate that property with an instance variable called _window
此时
[_window makeKeyAndVisible];
等同于
[self.window makeKeyAndVisible];
后者更推荐

如果在@interface里面写下
@property BOOL isFinishedFlag;
就等同于在@implementation里写下
@synthesize isFinishedFlag = _isFinishedFlag;

extern int 用来定义全局变量 Global Variables
static int 用来定义静态变量 Static Variables

enum 数据类型， 见207页
用来自定义数据类型

enum flag { false, true}; 用来定义一个数据类型「data type」flag
这个数据类型只能有false或true两个值
以flag这种数据类型来定义变量要用语句
enum flag endOfData, matchFound;
此时endOfData和matchFound的值只能为True或False

typedef 用自己的名字来等同于OC里的数据名称
比如，typedef Counter int;
可以把Counter等同为int，就可以直接用Counter来定义integer了。

如果一个计算中包含了float或int两个以上的计算类型，会自动进行数据类型转换，其中的一个会转化成另一个。见 P212

Bit操作符 & | ^ ~ << >> 见P213

用categories为一个class添加新的方法method

protocol是一个很多类共享的方法集合成的列表 P230
在protocol中列出的方法并没有包含实现implementation，需要等你来实现

例子
NSObject.h中包含的protocol有

@protocol NSCopying

  • (id)copyWithZone: (NSZone *)zone;
    @end

这个意味着如果你要把NSCopying用到你的类里，你必须写出copyWithZone的实现，你必须要在@interface里加上

@interface AddressBook: NSObject

如果要加超过一个protocol，需要写
@interface AddressBook: NSObject

Cocoa和iOS很依赖于protocol的方式，比如，如果要在iPhone里显示一个table，需要用UITableVIew类，但这个类并不知道table的标题，table包含了多少行和列。所以，UITableVIew类把这些责任给你，通过定义一个protocol叫做 UITableViewDataSource。当UITableVIew类需要信息，比如一个table里有多少列时，就会调用合适的方法(tableView:numberOfRowsInSection:)，你必须在你的类里根据protocol的定义来做定义。
详见 P233

定义一个class，里面包含了一个或多个其他类的对象，这种技术交Composite Objects 见P234
